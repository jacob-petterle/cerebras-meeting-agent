Resource Map → how Gemma observes the room

cerebras-meeting-agent · the in-call brain is an observer, not a chatbot · updated 2026-06-28

The one idea: Gemma never receives your conversation as a message. Each 5-second heartbeat, the loop reads the deltas of a few append-log resources and injects them as clearly-labeled <resource> blocks the model watches — then it picks exactly one tool (usually no_op). Your speech is just one resource among several. Nothing you say is ever "addressed to" the agent.

The resources we have

shape
{ participantId, senderKind: human|agent|tool, text, timestamp }
producers
mic → VAD → STT (human lines) · the tool write-back (agent + tool lines)
consumers
heartbeat delta-read · web console · the Gemma loop
INJECTED in the system channel as a <transcript> block of <utterance> elements (speaker · kind · ts) — framed "speakers talk to each other, not to you." Never a user turn.

shape
{ id, kind, title, description, filePath, … }

INJECTED as a <deliverables> resource block so Gemma can choose to share_screen a result by deliverableId. (Previously the loop only saw a text summary of it — now the artifact itself is a resource.)

The correction: conversation is a resource, not in-band

Before — in-band

messages: [
  { role: 'system', content: identity },
  { role: 'user',
    content: "[human:me] what's the weather" }
]

Structurally, Gemma is "the assistant being spoken to." Biases it to reply, and contradicts its own identity ("not a chatbot waiting to be addressed").

After — resource (system channel)

{ role: 'system', content: identity + resource convention },
{ role: 'system', content:
  "<current-time iso=… />
   <transcript since=0>
     <utterance speaker='me' kind='human'>what's the weather</utterance>
   </transcript>
   <deliverables> … </deliverables>" },
{ role: 'user', content: "[heartbeat] pick one action (or no_op)." }

Every observed resource — incl. the conversation — lives in the system channel. The only user turn is a content-free heartbeat pulse. Nothing the model reads as conversation is in-band. Mirrors Shipyard's renderEnvelope + the system-prompt that declares resources "the tracked truth… not a message addressed to you."

Why it matters

  • It makes the premise true. The product is a scoped participant that listens and chooses to help — not a chatbot you talk to. The mechanism now matches the pitch.
  • It generalizes. Every input the brain sees is a resource injected the same way — transcript, sub-agent deliverables, and (later, for Zoom) screen/participant state. One uniform seam.
  • It reduces reflexive chatter. Observed ambient state biases toward no_op; an addressed user turn biases toward answering. The whole design wants the former.
