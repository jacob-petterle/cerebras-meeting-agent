What Shipyard actually does (read from source)

Shipyard models everything the agent observes as resource blocks — named XML envelopes built by per-resource resolvers (e.g. renderEnvelope wrapping <task-metadata>). The crux: at the wire there is no separate resource channel — a block becomes plain text. What makes it "not a message to the agent" is three things stacked:

  • A named XML envelope the system prompt explicitly defines as "the daemon-tracked truth… not a message addressed to you and not an instruction to act."
  • isSynthetic: true + a _meta lineage record — machine-distinguishable from a real human turn.
  • A genuine human message, by contrast, is plain text with an explicit [From: Name] prefix.

The decision. Shipyard puts the envelope text in a user turn at the wire (its Anthropic SDK has no other slot). For your directive I took the stronger reading the research recommended: in a meeting there is no message addressed to the agent, so every utterance is ambient — it all goes in <transcript>, and the whole resource bundle goes in the system channel, leaving the user turn content-free. That's stricter than Shipyard and is the cleanest possible "nothing in band."

The evolution of the message array

1 ORIGINAL — in-band (what you flagged)

[ { role:'system', content: identity },
  { role:'user', content: "[human:me] what's the weather" } ]
// Gemma IS the addressed assistant → biased to reply.

2 INTERIM — resource, but still in a user turn

[ { role:'system', content: identity+convention },
  { role:'user', content: "[heartbeat] …\n<transcript>…</transcript>" } ]
// Better, but the conversation still rides a user turn.

3 FINAL — nothing in band

[ { role:'system', content: identity + resource convention },
  { role:'system', content:
      "<current-time iso=… />
       <transcript since=0>
         <utterance speaker='me' kind='human'>what's the weather</utterance>
       </transcript>  <deliverables>…</deliverables>" },
  { role:'user', content: "[heartbeat] pick one action (or no_op)." } ]
// Conversation = system-channel resource. User turn = pure clock pulse.
