Decisions

DECIDEDResources = seqNo append-log, NOT Loro

Why: deep-read of ~/repos/shipyard showed three "since X" systems. Shipyard stores messages/deliverables/tasks/comments as plain JSON/JSONL with seqNo = line index; Loro CRDT is used only for the plan rich-text doc + canvas. Our transcript + deliverables are single-writer append-only → a monotonic seqNo is the entire contract. Tradeoff: we drop CRDT merge / version-vectors we don't need; we keep wire shapes (subscribe{sinceSeqNo} / catch_up) compatible with Shipyard's, so a future daemon bridge is a transport swap, not a remodel.

Why: Cerebras is HTTP (openai npm), @cursor/sdk is native TS, and the media leg runs in Node. One runtime, shared types between brain + web. Gave up: parakeet-mlx (MLX/Python) → swapped for Moonshine on the CoreML EP (still ANE).

DECIDEDMedia: Moonshine STT + kokoro-js TTS, local-first

Why: verified in npm/source. Moonshine via @huggingface/transformers device:'coreml' (ANE, maintained); kokoro-js (Node CPU, ~sub-1s/sentence). Behind a swap interface → hosted Deepgram/Cartesia TS SDKs if local latency disappoints. Still all-TS either way.

DECIDEDPorts & adapters · Zoom is a late-bound swap

Why: the brain/tools/resource spine are transport-agnostic; the seam is raw PCM in / audio out / render-command out. Local mic/speakers/browser now; Zoom adapter later. Build core once. call_agent mocked first (fixed FINDINGS.html) so the spine is green before the real Cursor SDK.
